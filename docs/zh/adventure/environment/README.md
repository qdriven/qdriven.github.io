# 2020测试漫游 - 测试开发环境搭建

大部分的测试想学习写代码，遇到的第一件事情就是开发环境突然配置不起来了,代码运行不了了，
然后尝试了一点时间就放弃了. 由于现在开发中使用的技术种类相对来说还是不少，所以有时开发环境的问题确实经常会遇到。
希望通过下面几篇文章可以解决不少问题.

开发环境我个人总结主要分如下几种类型:
1. 语言运行环境搭建
2. 不同语言依赖包的安装和管理
3. 系统级别的一些依赖安装

个人的经验和实际遇到的问题可以总结如下几类:
1. 为什么按照教程安装之后，运行某个命令没有任何反应呢，或者直接报错了呢
2. 为什么依赖包安装不了呢
3. 为什么依赖包会有冲突呢
4. 常见的系统命令如何使用呢？


对于开发语言而言，目前不同的开发语言的套路越来越近似，一般都有:
1. 语言包，比如JDK，GOLANG，NODEJS，PYTHON的安装包
2. 依赖管理器比如JAVA的MAVEN/GRADLE，GO的GO MOD，PYTHON的pip，node的npm等等

所以下面的一系列文章主要针对这几种语言来说明如何解决这些问题，让自己搭建开发环境的时候不要被这些初级问题卡住。
同时也会介绍一些常用的IDE和插件，来帮助提高效率。

所以综合以上，主要通过分不同的语言在MAC和Windows上面进行介绍.
